# Projeto Integrador: Desenvolvimento Mobile

## Objetivo Geral

Solucionar um problema real para um cliente através de uma aplicação _**mobile**_ que envolva a integração entre banco
de dados, Backend e Frontend.

Para o projeto atual, nosso cliente trabalha com serviços de costura. A princípio nosso foco será na criação de um aplicativo para a **gestão de pedidos e clientes** do negócio de nosso usuário.

## Objetivos Específicos:

1. Trabalhar em equipe utilizando metodologias ágeis (Scrum) visando facilitar o desenvolvimento do projeto;
2. Aplicar conceitos de engenharia de requisitos para compreensão do problema, especificação e organização das
tarefas a serem realizadas;
3. Focar em realizar entregas (funcionalidades do sistema) que tragam valor ao cliente;
4. Focar na entrega de um MVP (Minimum Viable Product), deixando as funcionalidades menos importantes em
segundo plano;
5. Integrar diferentes tecnologias para desenvolvimento de uma aplicação cliente-servidor.

## Requisitos Mínimos

1. Persistência de dados: Desenvolver um back-end para persistir os dados ou utilizar os
serviços Firebase.
2. Aplicação com no mínimo 4 telas
3. Interfaces amigáveis (UX) para usuários comuns de acordo com o problema a ser resolvido.
4. Controle de acesso: login/senha e token de acesso;
5. O App Mobile pode ser desenvolvido com as tecnologias:
    - Híbrida: com React Native e expo.
    - Nativa: Android Studio com Kotlin + Jetpack Compose
6. Nome e ícone do App personalizados.
7. No mínimo uma operação no estilo CRUD completa (Cadastro, edição, listagem e exclusão)
8. Código em repositórios para controle de versionamento do sistema (GIT).

## Requisitos Opcionais

A implementação de requisitos opcionais pode garantir pontos extras na
avaliação. Seguem algumas sugestões:

- Mapas e geolocalização
- Integrar com uma aplicação web já existente
- Integrar com uma api de serviços de terceiros (clima, chat gpt, etc)
- Publicar o App nas lojas
